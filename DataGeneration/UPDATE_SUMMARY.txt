# DataGeneration Updates Summary

## Problem
The original DataGeneration code was only inserting data into the `bronze_leases` table, but the frontend application reads from the `silver_leases` table. This meant generated synthetic data wasn't appearing in the app.

## Solution
Updated the DataGeneration scripts to properly flow data from Bronze → Silver tables:

### Files Updated

1. **`generate_synthetic_leases.py`** - Enhanced to include:
   - `extracted_at` timestamp field (required for bronze schema)
   - Higher ratio of VERIFIED records (5:1 instead of 3:1)
   - Proper column ordering in INSERT statements

2. **`generate_and_promote.py`** - NEW all-in-one script that:
   - Generates synthetic lease data
   - Inserts into bronze_leases table
   - Automatically promotes VERIFIED records to silver_leases
   - Provides verification counts
   - Can accept command-line argument for number of leases

3. **`promote_to_silver.py`** - NEW standalone promotion script that:
   - Reads VERIFIED records from bronze_leases
   - Transforms and promotes them to silver_leases
   - Shows before/after counts
   - Can be run independently after bronze insertion

4. **`setup.sh`** - Updated to:
   - Reference the new `generate_and_promote.py` script
   - Prompt for number of leases
   - Pass the count as command-line argument

5. **`README.md`** - Enhanced with:
   - Quick start section highlighting `generate_and_promote.py`
   - Clear explanation of bronze → silver data flow
   - FAQ about why data wasn't appearing in frontend
   - Comparison of the three different scripts

6. **`QUICKSTART.md`** - Updated with:
   - Recommended all-in-one approach
   - Data flow diagram
   - Manual two-step process option
   - Troubleshooting for frontend data issues

## Key Changes

### Schema Alignment
- Added `extracted_at` field to match bronze_leases schema
- Updated INSERT statements to include all required columns in correct order

### Validation Status
- Changed ratio from 75% VERIFIED to 83% VERIFIED (5 out of 6)
- Ensures more data flows automatically to silver layer

### Complete Pipeline
- New script handles end-to-end flow automatically
- No manual intervention needed between bronze and silver

## Data Flow

```
generate_and_promote.py
   ↓
Generate synthetic leases (in memory)
   ↓
Insert into bronze_leases table
   ↓
Filter VERIFIED records
   ↓
Transform and MERGE into silver_leases table
   ↓
Frontend reads from silver_leases
```

## Usage

### Recommended (All-in-One)
```bash
python3 generate_and_promote.py 100
```

### Alternative (Two Steps)
```bash
python3 generate_synthetic_leases.py
python3 promote_to_silver.py
```

## Benefits

1. **Immediate Frontend Visibility** - Data appears in app right away
2. **Automatic Promotion** - No manual SQL queries needed
3. **Verification Built-in** - Shows counts at each stage
4. **Flexible Options** - Can use all-in-one or step-by-step approach
5. **Better Documentation** - Clear explanation of data flow
